# Data Model: Account and Event Creation

All identifiers are MongoDB ObjectIds internally and exposed as opaque GraphQL IDs. Every document has
`createdAt`, `updatedAt`, and `schemaVersion`. Dates are UTC instants. Raw credentials and link secrets are
never stored.

## Account

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated by service |
| `emailNormalized` | string | Required, trimmed/lowercase, unique; never returned publicly |
| `passwordHash` | string | Required Argon2id encoded hash; never returned |
| `verificationStatus` | enum | `pending` or `verified` |
| `verifiedAt` | date/null | Set once verification succeeds |
| `credentialVersion` | integer | Increment to invalidate sessions after credential changes |
| `createdAt`, `updatedAt` | date | Service managed |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: unique `{ emailNormalized: 1 }`.

**Transitions**: `pending -> verified`. Verification cannot be reversed in this feature.

## EmailVerification

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `accountId` | ObjectId | Required account reference |
| `tokenDigest` | string | Required, unique SHA-256 digest with server pepper |
| `expiresAt` | date | Required; expired tokens cannot be consumed |
| `consumedAt` | date/null | Set atomically on first valid use |
| `createdAt` | date | Service managed |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: unique `{ tokenDigest: 1 }`; TTL `{ expiresAt: 1 }`; `{ accountId: 1, createdAt: -1 }`.

**Transitions**: `active -> consumed` or `active -> expired`. Issuing a replacement invalidates prior active
records for that account.

## Session

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `accountId` | ObjectId | Required account reference |
| `secretDigest` | string | Required, unique digest; cookie contains raw secret |
| `credentialVersion` | integer | Must match current account value |
| `lastSeenAt` | date | Throttled updates avoid write-on-every-request |
| `expiresAt` | date | Absolute expiry |
| `revokedAt` | date/null | Set on sign-out/invalidation |
| `createdAt` | date | Service managed |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: unique `{ secretDigest: 1 }`; TTL `{ expiresAt: 1 }`; `{ accountId: 1, revokedAt: 1 }`.

**Transitions**: `active -> revoked` or `active -> expired`. No reverse transition.

## Event

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `ownerAccountId` | ObjectId | Required and immutable; derived from session |
| `publicId` | string | Required, unique, unguessable URL-safe identifier |
| `title` | string | Required after trim; 1–120 characters |
| `description` | string/null | Optional; maximum 2,000 characters |
| `location` | string/null | Optional; maximum 300 characters |
| `visibility` | enum | `private` (default) or `public` |
| `createdAt`, `updatedAt` | date | Service managed |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: unique `{ publicId: 1 }`; `{ ownerAccountId: 1, createdAt: -1 }`.

**Transitions**: visibility may change `private <-> public` only by the owner. Access is recalculated on
every read; existing access records remain dormant while public and apply again if returned to private.

## EventAccess

One record represents one person's private access to one event. The email identifies a pending invite;
`accountId` binds it after the matching verified user accepts or next accesses the event.

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `eventId` | ObjectId | Required event reference |
| `emailNormalized` | string | Required; visible only to the host |
| `accountId` | ObjectId/null | Must resolve to the same normalized verified email |
| `status` | enum | `invited`, `active`, `revoked`, `expired` |
| `invitationDigest` | string/null | Unique digest while invitation is usable |
| `invitationExpiresAt` | date/null | Required while `invited` |
| `acceptedAt` | date/null | Set when access becomes active |
| `revokedAt` | date/null | Set by host removal |
| `createdByAccountId` | ObjectId | Must equal event owner at creation |
| `createdAt`, `updatedAt` | date | Service managed |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: unique partial `{ eventId: 1, emailNormalized: 1 }` for non-deleted records; unique partial
`{ invitationDigest: 1 }`; `{ accountId: 1, status: 1 }`; TTL cleanup uses a separate expiration marker only
after a record no longer needs audit retention.

**Transitions**: `invited -> active`, `invited -> expired`, and `invited|active -> revoked`. Reinviting a
revoked/expired address issues a new secret and returns the record to `invited` while preserving audit
events.

## IdempotencyRecord

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `scope` | string | Account ID or normalized registration scope |
| `operation` | string | Mutation name |
| `key` | string | Client-generated UUID |
| `requestDigest` | string | Reject reuse with different input |
| `resultReference` | object | Minimal stable reference to created resource/result |
| `expiresAt` | date | Retention appropriate to retry window |
| `createdAt` | date | Service managed |

**Indexes**: unique `{ scope: 1, operation: 1, key: 1 }`; TTL `{ expiresAt: 1 }`.

## AuditEvent

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `name` | string | Stable event name |
| `actorAccountId` | ObjectId/null | Null for anonymous/system actions |
| `subjectType`, `subjectId` | string | Target without embedding private payloads |
| `outcome` | enum | `success`, `denied`, `failure` |
| `correlationId` | string | Connects audit event to request logs |
| `metadata` | object | Allowlisted non-secret context only |
| `createdAt` | date | Immutable |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: `{ subjectType: 1, subjectId: 1, createdAt: -1 }`; `{ actorAccountId: 1, createdAt: -1 }`;
`{ correlationId: 1 }`.

## Relationships and Invariants

- Account 1-to-many Session, Event, EmailVerification, and accepted EventAccess records.
- Event belongs to exactly one Account and has zero-to-many EventAccess records.
- Event ownership is never supplied by a client and cannot change in this MVP.
- A public Event is readable by anyone with `publicId`. A private Event is readable only by its owner or
  an authenticated account with an `active` EventAccess.
- An Account can activate access only when its normalized verified email matches the access record.
- Deleting accounts/events is out of scope; repositories do not expose deletion operations for them.
