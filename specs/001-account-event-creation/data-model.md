# Data Model: Account, Event, and Participant Registration

All identifiers are MongoDB ObjectIds internally and exposed as opaque GraphQL IDs. Every document has
`createdAt`, `updatedAt`, and `schemaVersion`. Dates are UTC instants. Raw credentials and link secrets are
never stored.

## Account

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated by service |
| `emailNormalized` | string/null | Normalized, unique when present; required for completed MVP accounts |
| `phoneNormalized` | string/null | E.164-normalized, unique when present; allowed for provisional accounts |
| `referredByAccountId` | ObjectId/null | Host account that added this account as a participant; null for self-registered accounts |
| `lifecycleStatus` | enum | `provisional` or `completed` |
| `passwordHash` | string/null | Required Argon2id hash only for completed accounts; never returned |
| `verificationStatus` | enum | `pending` or `verified` |
| `verifiedAt` | date/null | Set once verification succeeds |
| `credentialVersion` | integer | Increment to invalidate sessions after credential changes |
| `createdAt`, `updatedAt` | date | Service managed |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: unique partial `{ emailNormalized: 1 }` when non-null; unique partial
`{ phoneNormalized: 1 }` when non-null.

**Transitions**: Visitor-created accounts move `pending -> verified` and become completed with a null
`referredByAccountId`. Host-created participant accounts remain `provisional` and `pending`, store the
creating host's account ID in `referredByAccountId`, and keep claiming or completing them out of scope.

## EmailVerification

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `accountId` | ObjectId | Required account reference |
| `tokenDigest` | string | Required, unique SHA-256 digest with server pepper |
| `expiresAt` | date | Required; expired tokens cannot be consumed |
| `consumedAt` | date/null | Set atomically on first valid use |
| `createdAt` | date | Service managed |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: unique `{ tokenDigest: 1 }`; TTL `{ expiresAt: 1 }`; `{ accountId: 1, createdAt: -1 }`.

**Transitions**: `active -> consumed` or `active -> expired`. Issuing a replacement invalidates prior active
records for that account.

## Session

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `accountId` | ObjectId | Required account reference |
| `secretDigest` | string | Required, unique digest; cookie contains raw secret |
| `credentialVersion` | integer | Must match current account value |
| `lastSeenAt` | date | Throttled updates avoid write-on-every-request |
| `expiresAt` | date | Absolute expiry |
| `revokedAt` | date/null | Set on sign-out/invalidation |
| `createdAt` | date | Service managed |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: unique `{ secretDigest: 1 }`; TTL `{ expiresAt: 1 }`; `{ accountId: 1, revokedAt: 1 }`.

**Transitions**: `active -> revoked` or `active -> expired`. No reverse transition.

## Event

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `ownerAccountId` | ObjectId | Required and immutable; derived from session |
| `publicId` | string | Required, unique, unguessable URL-safe identifier |
| `title` | string | Required after trim; 1–120 characters |
| `description` | string/null | Optional; maximum 2,000 characters |
| `location` | string/null | Optional; maximum 300 characters |
| `registrationPolicy` | enum | `admin_managed` (default) or `open` |
| `createdAt`, `updatedAt` | date | Service managed |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: unique `{ publicId: 1 }`; `{ ownerAccountId: 1, createdAt: -1 }`.

**Transitions**: registration policy may change `admin_managed <-> open` only by the owner. Event details
remain viewable to anyone with the direct link under either policy.

## EventRegistration

One record represents one account's participation in one voting event. The account may be completed or a
host-created provisional account. Registration does not grant event-view permission because direct-link
viewing is always available.

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `eventId` | ObjectId | Required event reference |
| `accountId` | ObjectId | Required immutable account reference, including provisional accounts |
| `status` | enum | `registered` or `removed` |
| `registrationSource` | enum | `self` or `host` |
| `registeredByAccountId` | ObjectId | Self account for OPEN registration or event owner for host registration |
| `removedAt` | date/null | Set by host removal |
| `createdAt`, `updatedAt` | date | Service managed |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: unique `{ eventId: 1, accountId: 1 }`; `{ accountId: 1, status: 1 }`;
`{ eventId: 1, status: 1, createdAt: 1 }`.

**Transitions**: `registered -> removed`; re-adding a removed participant returns the same record to
`registered` while preserving audit events. No verification or invitation is emitted for host-created
provisional accounts.

## IdempotencyRecord

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `scope` | string | Account ID or normalized registration scope |
| `operation` | string | Mutation name |
| `key` | string | Client-generated UUID |
| `requestDigest` | string | Reject reuse with different input |
| `resultReference` | object | Minimal stable reference to created resource/result |
| `expiresAt` | date | Retention appropriate to retry window |
| `createdAt` | date | Service managed |

**Indexes**: unique `{ scope: 1, operation: 1, key: 1 }`; TTL `{ expiresAt: 1 }`.

## AuditEvent

| Field | Type | Rules |
|---|---|---|
| `_id` | ObjectId | Generated |
| `name` | string | Stable event name |
| `actorAccountId` | ObjectId/null | Null for anonymous/system actions |
| `subjectType`, `subjectId` | string | Target without embedding private payloads |
| `outcome` | enum | `success`, `denied`, `failure` |
| `correlationId` | string | Connects audit event to request logs |
| `metadata` | object | Allowlisted non-secret context only |
| `createdAt` | date | Immutable |
| `schemaVersion` | integer | Starts at 1 |

**Indexes**: `{ subjectType: 1, subjectId: 1, createdAt: -1 }`; `{ actorAccountId: 1, createdAt: -1 }`;
`{ correlationId: 1 }`.

## Relationships and Invariants

- Account 1-to-many Session, Event, EmailVerification, and EventRegistration records.
- Event belongs to exactly one Account and has zero-to-many EventRegistration records.
- Event ownership is never supplied by a client and cannot change in this MVP.
- Every Event is readable by anyone with `publicId`.
- An `open` Event permits a verified account to register itself. An `admin_managed` Event permits only its
  owner to register participants.
- Host registration requires email and accepts an optional phone; it reuses an Account matching a supplied normalized identifier or atomically creates a
  provisional unverified Account, stores the host's account ID in `referredByAccountId`, and immediately
  references the participant account's ID from EventRegistration.
- Deleting accounts/events is out of scope; repositories do not expose deletion operations for them.
